# 자동 커밋 및 푸시

현재 변경 사항을 자동으로 커밋하고 푸시합니다.

## 실행 단계

1. `git status`로 변경 사항 확인
2. `git diff`로 변경 내용 분석
3. 적절한 한글 커밋 메시지 생성 (feat:, fix:, style: 등)
4. 변경 사항 스테이징 및 커밋
5. 현재 브랜치에 푸시 (실패 시 4회 재시도)

## 커밋 메시지 규칙

- `feat:` - 새로운 기능 추가
- `fix:` - 버그 수정
- `style:` - 코드 스타일/포맷 변경
- `docs:` - 문서 변경
- `refactor:` - 리팩토링
- `test:` - 테스트 추가/수정
- `chore:` - 기타 변경
